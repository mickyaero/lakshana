The wedge

Every AI document-extraction tool assumes you already know your schema. Real-world data is messier than that — you get a hard drive of receipts, contracts, statements, and ???, and you have to figure out the shape before you can extract anything.

Lakshana is the step before extraction. The "tell me what I have" step that every document pipeline skips, and pays for later.

— what this library does, in one paragraph

How it works

Five steps. Each one falls back gracefully. Each output is from real text, not a hallucination.

1
Parse
PDFs, images, text. Embedded text first, OCR fallback.
2
Embed
Hybrid signal — structural fingerprint + semantic.
3
Cluster
UMAP projects, HDBSCAN finds the doc types.
4
Infer
An LLM proposes fields with names, types, examples.
5
Verify
Each field is re-checked across the cluster with grounding quotes.

Install

One command. Bring whichever LLM you have a key for.

# live on PyPI — pick whichever provider you have a key for
$ pip install "lakshana[openai]"      # also serves Groq, Cerebras, OpenRouter, Ollama
$ pip install "lakshana[anthropic]"   # Claude
$ pip install "lakshana[google]"      # Gemini
$ pip install "lakshana[all]"         # everything

# Python API — works with whichever LLM you have a key for
>>> from lakshana import discover
>>> result = discover(files=[...], model="openai/gpt-4o-mini")
>>> for c in result.clusters:
...     print(c["name"], result.schemas[str(c["id"])])

# CLI
$ lakshana analyze ./docs --output schema.json
$ lakshana analyze ./docs --model claude-sonnet-4-6 --output schema.json
$ lakshana analyze ./docs --model openai/gpt-4o-mini --output schema.json

Why it exists

I built it because I needed it, and nothing else did this.

I spent the last year building an internal document-extraction stack focused on Indian financial documents. The hardest part was never the extraction itself. It was figuring out, for any new client, what their documents even were.

So I pulled out the piece that did the figuring-out and put it on its own. Lakshana is what I wish existed when I started.

If you're building anything in this space, I'd love to hear from you — open an issue with what you're trying to do.
